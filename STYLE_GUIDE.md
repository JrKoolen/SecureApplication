# Style Guide

## General Guidelines

- **No emojis** in code, comments, documentation, or any project files
- Use plain text for all communication and documentation
- Keep code clean and professional

## Code Style

- Follow TypeScript best practices
- Use meaningful variable and function names
- Add comments where necessary (without emojis)
- Maintain consistent formatting

## Documentation

- Use plain text only
- No emojis in README, comments, or any documentation
- Keep documentation clear and professional
